My Lab Works

Lab 1: UI Layout Magic

Building a neat and clean static user interface structure using Jetpack Compose by organizing core layout components such as Row, Column, and Box systematically.

My Thoughts: My very first experiment with the declarative UI framework! While structuring the app text and vector icons felt slightly challenging at first, it firmly taught me the fundamentals of responsive and flexible mobile grid systems.

Lab 2: Making Things Interactive

Implementing interactive app logic such as actionable buttons, user text fields, and managing dynamic states variables using mutableStateOf wrapper flows.

My Thoughts: Learning local state management truly opened my eyes to how data flows reactively in real-time. Developing an interactive cup counter that immediately refreshes the UI text gave me an incredible sense of coding satisfaction.

Lab 3: Material Styling Sparkles

Enhancing the aesthetic design of the application in accordance with the Material Design 3 (MD3) guidelines by incorporating custom color palettes, modern typography styling, and smooth curved shapes.

My Thoughts: This lab completely transformed my application from a basic prototype into an expressive, professional ecosystem. Utilizing soft pastel green tones blended perfectly with the health and well-being message I intended to deliver.

Lab 4: Journey Maps & Navigation

Connecting distinct app screens seamlessly using the Jetpack Navigation Controller framework alongside a structured bottom navigation system for effortless switching.

My Thoughts: Routing screen arguments across multiple destinations initially challenged my algorithmic logic. However, successfully animating transitions from the profile screen to the primary tracking board made the system feel inherently user-centric.

Lab 5: Smart Local Storage (Room)

Architecting a reliable local data persistence engine using Room Database (SQLite layer) to seamlessly store user hydration history records for offline availability.

My Thoughts: Gaining concrete insights into structural data design via DAOs and abstract entities was a major milestone. Knowing that vital user logs remain intact even after terminating the application adds exceptional robustness.

Special Projects

🚀 Stay Hydrate

The Problem: Many busy individuals completely forget to drink sufficient water throughout the day, often leading to fatigue, lack of mental focus, and long-term health complications.

An interactive daily water tracking utility that allows users to set customizable fluid intake milestones, record precise consumption sessions periodically, and monitor visual target metrics using rich progress indicators.

Lessons Learned: This launch served as the ultimate sandbox to combine all my isolated lab methodologies. I fully mastered binding data pipelines through ViewModels to preserve application consistency during layout adjustments and configuration changes.

🌟 Stay Hydrate v2

The Problem: Isolated intake logging makes it tedious for users to monitor holistic lifestyle choices. Users need a centralized assistant that connects their hydration metrics directly with daily nutritional food tracking.

An advanced version of the app upgraded with real-time camera scanning capabilities to identify meal/drink types, integrated calorie calculations, sleek trend analysis graphs for tracking monthly habits, and secure cloud storage integration driven by Firebase.

Lessons Learned: Building this second version provided immense industry-grade exposure in marrying physical device camera feeds with cloud infrastructures (Firebase Authentication & Realtime Database).
